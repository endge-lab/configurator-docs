# События и обновления

В Endge важно различать контракт события, конкретное возникновение события,
прикреплённую реакцию и глобальную шину сообщений. Это четыре разные границы.

## Event contract

`definePorts.emits` описывает публичный типизированный порт Component SFC. Он
хранится в Source, проверяется compiler-ом и попадает в program artifact.
Контракт отвечает на вопрос: «какое событие этот компонент может публиковать и
какой у него payload?»

```ts
const ports = definePorts({
  emits: {
    detailsOpened: event<{ id: string }>(),
  },
})
```

Объявление контракта само по себе не является occurrence.

## Runtime occurrence

Occurrence — конкретный факт в конкретном смонтированном экземпляре. Renderer
или системный producer вызывает границу Event runtime-host-а. Host предоставляет
методы `emitEventPort(name, payload)` и `onEventPort(name, listener)`.

У каждого вложенного Component SFC своя граница. Она запускает реакцию текущего
компонента, применяет `forward` и передаёт occurrence родительской границе.
Цепочка ограничена 32 переходами.

Built-in tag может быть непосредственным producer-ом occurrence. Локальный
`@click="action(...)"` запускает реакцию в границе текущего Component SFC, но не
создаёт публичный контракт автоматически. Публичной частью occurrence становится
только через `definePorts.emits.from` или `forward`.

Modifier `.stop` имеет двойную renderer-neutral семантику: renderer прекращает
нативный bubbling, а Event boundary не передаёт occurrence дальше. Локальная
реакция при этом выполняется. `.prevent`, `.self`, `.once`, `.capture` и
`.passive` применяются renderer-ом; DOM Event за границу renderer-а не выходит.

Полный список intrinsic Events, payload-контракты и практические варианты
объявления находятся в [руководстве по событиям Component SFC](/guides/component-events).

## Прикреплённый Action

Поле `action` в `event({...})` — текущая исполняемая реакция Source. Это не
отдельный binding-документ и не обработчик, который поглощает событие.

Порядок смысловых операций:

1. Event публикуется подписчикам текущей границы.
2. Runtime запускает `action`, если он указан.
3. Event продолжает `forward` в родительский компонент.
4. Ошибка Action записывается в diagnostics и не отменяет пункты 1 и 3.

Пользователь может заменить или удалить `action` в визуальном редакторе. При
удалении контракт `emits` сохраняется.

## Почему это не `Endge.events`

`Endge.events` — глобальная transport-шина для системных runtime-сообщений. Она
не является реестром Component SFC contracts и не получает автоматически каждое
возникновение UI Event. Иначе одинаковые локальные компоненты создавали бы
глобальный шум и теряли бы границы экземпляров.

Если интеграции действительно нужна глобальная публикация, она должна быть
явным Action или отдельным bridge, выбранным автором приложения.

## Связь с Raph, Updates и SSE

Raph доставляет изменения данных до runtime-узлов. Event сообщает о смысловом
факте. Изменение данных может вызвать Event, но эти механизмы не заменяют друг
друга.

`Endge.sse` принимает сообщения сервера, а `Endge.updates` проверяет и применяет
разрешённые обновления. Только после применения обновление может привести к
реактивному изменению или смысловому Event.
