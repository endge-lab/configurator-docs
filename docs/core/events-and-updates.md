# События и обновления

`Endge.events` — transport-слой для доставки runtime-сообщений подписчикам из кода. Он не является реестром контрактов и не хранит bindings.

## Три разных понятия

- **EventContract** описывает допустимое событие и его payload.
- **Binding** хранит конфигурируемое правило реакции.
- **Endge.events** доставляет уже возникшее сообщение кодовым подписчикам.

## Типичный поток

1. Runtime-host фиксирует смысловой факт.
2. Ядро нормализует его как каноническое событие.
3. Binding dispatcher находит конфигурируемые реакции.
4. То же событие может быть опубликовано в `Endge.events` для кода.

## Базовый API

```ts
const unsubscribe = Endge.events.onEvent('query:completed', event => {
  console.log(event.payload)
})

Endge.events.emitEvent('query:completed', {
  queryId: 'items-query',
})

unsubscribe()
```

Для динамических имён используются `onDynamic` и `emitDynamic`. Подписка всегда должна освобождаться владельцем, создавшим её.

## Связь с Raph

Raph доставляет изменения данных до runtime-узлов. `Endge.events` доставляет смысловые сообщения до кодовых подписчиков. Изменение данных может привести к событию, но эти механизмы не заменяют друг друга.

## Updates и SSE

`Endge.sse` получает сообщения сервера, а `Endge.updates` применяет разрешённые обновления. После применения могут возникнуть канонические события и реактивные изменения. Входящее сообщение нельзя считать применённым обновлением до его проверки и обработки.
