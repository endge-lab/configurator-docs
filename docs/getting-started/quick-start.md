# Быстрый старт

Этот сценарий показывает общий порядок работы без привязки к конкретному проекту.

## 1. Выберите контекст

Откройте проект и нужное окружение. Окружение определяет переменные, профили авторизации и другие значения, которые отличаются между разработкой и эксплуатацией.

## 2. Изучите дерево домена

В дереве находятся документы проекта. Откройте существующий документ, чтобы увидеть его source, метаданные и доступные действия.

## 3. Создайте документ

Выберите тип сущности и задайте уникальный identity. Для первого знакомства удобно начать с Type, Query или ComponentSFC.

## 4. Отредактируйте source

Редактор показывает исходное описание документа. Сохраните изменения только после того, как синтаксис прошёл проверку.

## 5. Проверьте результат

Используйте доступный preview, инспектор или диагностику. Ошибки компиляции относятся к source, а ошибки выполнения — к конкретному runtime-запуску.

## 6. Свяжите документы

Добавьте входы и ссылки на другие сущности. Проверяйте не только каждый документ отдельно, но и полный поток данных.
