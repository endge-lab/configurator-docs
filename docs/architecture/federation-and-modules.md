# Федерации и модули

Федерация позволяет приложению подключать модули с собственными документами, runtime-расширениями и UI-регистрациями без слияния всей реализации в один пакет.

## Модуль

Модуль должен явно объявлять:

- identity и версию;
- зависимости;
- экспортируемые доменные документы;
- compiler/runtime extensions;
- UI definitions и presentation adapters;
- bootstrap и dispose hooks.

## Установка

Рекомендуемый порядок:

1. проверить совместимость версии и зависимости;
2. зарегистрировать статические контракты;
3. загрузить или объединить разрешённые документы домена;
4. выполнить runtime bindings;
5. подключить UI adapter и integrations;
6. зафиксировать module handle для последующего dispose.

## Изоляция

Модуль не должен изменять чужие коллекции или глобальное состояние без публичного контракта. Регистрации должны быть обратимыми, а identity — иметь предсказуемую область уникальности.

## Федерация

Федерация разрешает набор модулей для конкретного приложения или tenant. Она отвечает за composition и policy, но не переписывает внутренние контракты модулей.

Это позволяет держать ядро небольшим и подключать функциональность по потребности.
