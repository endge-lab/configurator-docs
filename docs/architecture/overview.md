# Общая архитектура Endge

Endge разделяет авторское описание, компиляцию, исполнение и отображение. Это позволяет одному домену работать с разными приложениями и presentation adapters.

```text
Configurator / source files
          ↓
       Domain
          ↓
       Compiler
          ↓
   Program artifacts
          ↓
       Runtime
          ↓
Presentation adapter / integrations
```

## Слои

### Authoring

Конфигуратор и source-файлы помогают создавать документы. Их UI-состояние не является частью исполняемого контракта.

### Domain

Домен хранит документы и связи по identity. Он должен оставаться сериализуемым и пригодным для версионирования.

### Compiler и Program

Compiler проверяет source и создаёт нормализованные artifacts. Program — явная граница между сохранённым описанием и исполнением.

### Runtime

Runtime создаёт живые hosts, подключает реактивные данные, события и ресурсы. Runtime-состояние имеет владельца и lifecycle.

### Presentation

UI adapter материализует renderer-neutral результат в конкретной технологии. Его внутренние DOM-классы или компоненты не должны попадать в основной доменный контракт.

## Инварианты

- Source — единственная авторская истина.
- Compiler не зависит от открытого редактора.
- Program не хранит временное runtime-состояние.
- Runtime не переписывает source как побочный эффект.
- Presentation можно заменить без изменения смысла программы.
