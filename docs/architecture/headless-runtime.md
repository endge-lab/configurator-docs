# Headless Runtime

Runtime должен исполнять программу без обязательного визуального слоя. Presentation подключается только там, где результат действительно нужно показать.

## Зачем это нужно

- Query, Computation и Action можно запускать в фоне.
- Один program можно использовать в UI, worker или service-сценарии.
- Тестирование бизнес-логики не требует DOM.
- UI adapter не становится владельцем данных и lifecycle ядра.

## Optional Presentation

Presentation adapter получает уже готовый runtime-контракт. Он может отображать value, loading и error, публиковать пользовательские события и преобразовывать нейтральные стили в доступный backend.

Если presentation отсутствует, runtime всё равно должен корректно создать ресурсы, выполнить работу и освободить их.

## Граница ответственности

| Runtime | Presentation |
| --- | --- |
| исполняет program | отображает состояние |
| владеет ресурсами и подписками | владеет UI-экземпляром |
| публикует нейтральные события | переводит пользовательский ввод |
| не требует DOM | использует возможности конкретной среды |

Renderer-specific оптимизации допустимы внутри adapter, но не должны менять смысл source или program.
