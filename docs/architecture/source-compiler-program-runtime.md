# Source → Compiler → Program → Runtime

Эта цепочка задаёт основную границу исполняемых сущностей Endge.

## Source

Source — сохраняемый текст или структурированный документ, который редактирует пользователь. Он должен быть понятен человеку, стабильно сериализоваться и не содержать случайное состояние конкретного запуска.

## Compiler

Compiler выполняет четыре задачи:

1. разбирает source;
2. проверяет синтаксис и ссылки;
3. нормализует допустимые варианты записи;
4. создаёт program artifact и diagnostics.

Ошибка compiler должна ссылаться на исходный документ и по возможности на конкретное место source.

## Program

Program — immutable результат успешной компиляции. Он содержит достаточный контракт для исполнения и может включать child artifacts, например локальный DataView внутри Query.

Program не должен содержать Vue-компоненты, открытый editor state или живые подписки.

## Runtime

Runtime получает program и контекст запуска, создаёт host и управляет его lifecycle. Один program может породить несколько runtime instances с разными входами.

## Почему граница обязательна

Без Program runtime вынужден читать авторский source и повторять compiler-логику. Это смешивает ответственность, усложняет диагностику и делает результат зависимым от конкретного host. Явный artifact устраняет эту двойственность.
