# Рабочая область Runtime Preview

В конфигураторе используется единая область визуализации runtime — **Runtime Tree**. Отдельные плавающие превью для SFC, Composition и Demonstration больше не входят в рабочую область редактора.

## Сценарий работы

1. Откройте документ, который можно запустить в runtime: Project, Composition, Component SFC или Store.
2. Нажмите кнопку запуска либо `Command+Enter` / `Control+Enter`.
3. Редактор запустит текущий документ в Runtime Preview и активирует виджет Runtime Tree.
4. Выберите runtime-узел, чтобы посмотреть его жизненный цикл, результат отображения и входные props.
5. Нажмите `Escape`, чтобы активировать виджет Project и вернуться в рабочую область документа.

Возврат к Project не останавливает runtime. Управление его жизненным циклом выполняется в Runtime Tree.

## Черновики и сохранённые источники

При запуске Component SFC, Composition и Store используется текущее состояние редактора, включая несохранённые изменения. Документ не сохраняется автоматически. Для Project используется текущая скомпилированная модель проекта, поскольку у Project нет отдельного контракта компиляции source.

Такой подход позволяет безопасно проверять изменения кнопкой запуска, а все средства визуализации и управления жизненным циклом остаются в единой рабочей области Runtime Preview.

## Контракт интерфейса

- Для запуска используется одно действие Run; отдельного действия Debug нет.
- Run всегда активирует Runtime Tree.
- Runtime Tree отвечает за выбор runtime, действия жизненного цикла и подробности превью.
- `Escape` выполняет навигацию между рабочими областями и не уничтожает runtime.
- Контекстное меню или диалоговое окно может обработать первое нажатие `Escape`. После его закрытия следующее нажатие `Escape` возвращает пользователя к Project.
