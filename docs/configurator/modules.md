# Модули конфигуратора

Конфигуратор состоит из рабочих областей. Это части интерфейса, а не доменные сущности и не модули `Endge.*`.

## Основные области

| Область | Назначение |
| --- | --- |
| Домен | Навигация по документам проекта, создание, группировка и открытие сущностей |
| Редактор | Изменение source или структурированных полей выбранного документа |
| Инспектор | Метаданные, связи и технические свойства текущего объекта |
| Preview | Проверка компонента, композиции или другого исполняемого документа |
| Версии и хранилище | Работа со снимками, локальными данными и состоянием сохранения |
| Ошибки и события | Просмотр ошибок, событий и результатов диагностики |
| Runtime Debug | Наблюдение за runtime-host, реактивными узлами и жизненным циклом |
| Терминал и консоль | Выполнение разрешённых диагностических команд и просмотр журналов |
| Помощник | Подготовка предложений по изменениям домена через безопасный changeset-процесс |
| Справка | Открытие документации по пользовательским и техническим сценариям |

## Виджеты

Часть областей реализована как виджеты рабочей среды. Их можно показывать в подходящей панели, но назначение остаётся тем же. Виджет не становится отдельной сущностью домена только потому, что имеет собственное состояние интерфейса.

## Как выбрать нужную область

- Для создания и поиска документа начните с **Домена**.
- Для изменения документа перейдите в **Редактор**.
- Для проверки поведения используйте подходящий **Preview**.
- Для поиска причины ошибки откройте **Ошибки**, **События** и **Runtime Debug**.
- Для деталей синтаксиса перейдите в раздел [«Синтаксис и контракты»](/reference/query).

В дальнейшем документация отдельных сценариев может ссылаться на эти области, не создавая отдельную страницу для каждого виджета.
