# Консоль разработчика

В глобальную переменную `Endge` (в консоли браузера) выводится только объект с зарегистрированными командами. Класс Endge и внутренние модули (domain, debug, runtimeDebugger и т.д.) в глобал не публикуются - в консоли доступны только те методы, которые явно зарегистрированы.

## Регистрация обработчиков (и вызов в консоли)

Регистрация выполняется через `Endge.console` (доступен в коде приложения, не в глобальной консоли). После вызова `exposeToGlobal()` зарегистрированная команда появляется в консоли как метод `Endge.commandName()`.

### API

- **`Endge.console.register(name, fn)`** - регистрирует команду. `name` - строка (имя метода в консоли), `fn` - функция `(...args: any[]) => any`.
- **`Endge.console.unregister(name)`** - удаляет команду.
- **`Endge.console.exposeToGlobal()`** - выставляет в `globalThis.Endge` объект только с текущими зарегистрированными командами. Вызывать после регистрации (или после добавления новых команд).

### Как вызываются в консоли

После регистрации, например, `Endge.console.register('myCommand', () => { ... })` и вызова `Endge.console.exposeToGlobal()` в консоли разработчика становится доступен вызов:

```text
Endge.myCommand()
```

Имя команды совпадает с переданным в `register(name, ...)`. Аргументы в `fn` могут быть любыми - при вызове из консоли их можно передавать так же: `Endge.myCommand(arg1, arg2)`.

### Пример (в коде приложения или подмодуля)

```ts
import { Endge } from '@endge/core'

// Регистрация
Endge.console.register('ping', () => console.log('pong'))
Endge.console.register('greet', (name: string) => console.log('Hello,', name ?? 'world'))

// Проброс в глобал (консоль браузера получит Endge.ping() и Endge.greet())
Endge.console.exposeToGlobal()
```

В консоли разработчика:

```text
Endge.ping()        // → pong
Endge.greet('User') // → Hello, User
```

## Стандартные обработчики

Команды, которые регистрируются по умолчанию при инициализации приложения или админки.

### debugTab

**Где регистрируется:** при инициализации приложения (AppCore).

**Назначение:** подключает текущую вкладку к каналу отладки Runtime Debug. После вызова вкладка появляется в виджете «Runtime Debug» в админке, и её можно выбрать для отладки.

**Вызов в консоли:**

```text
Endge.debugTab()
```

### tab (админка)

**Где регистрируется:** при инициализации модуля админки (EndgeIDEConsole).

**Назначение:** выводит в консоль информацию о текущей открытой вкладке админки (активная вкладка, метаданные).

**Вызов в консоли:**

```text
Endge.tab()
```

### document (админка)

**Где регистрируется:** при инициализации модуля админки (EndgeIDEConsole).

**Назначение:** выводит в консоль данные текущего открытого документа: доменную модель (DocumentModel) и модель редактора (EditorModel). Удобно для отладки структуры документа во вкладке редактора.

**Вызов в консоли:**

```text
Endge.document()
```
