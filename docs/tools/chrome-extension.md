# Расширение Chrome

Расширение нужно для ручного сценария:

1. Открыта вкладка платформы.
2. Локально запущен codegen listener.
3. По клику в popup расширение забирает текущий домен из вкладки.
4. Доменные данные отправляются в локальную утилиту.
5. Утилита генерирует файлы в `src/gen`.

## Сборка

В корне проекта:

```bash
pnpm chrome-extension:build
```

Собранное unpacked-расширение лежит в:

```text
packages/@endge-chrome-extension/dist
```

## Установка в Chrome

1. Открой `chrome://extensions`
2. Включи `Developer mode`
3. Нажми `Load unpacked`
4. Выбери папку `packages/@endge-chrome-extension/dist`

После этого можно закрепить расширение через иконку пазла.

## Как понять, что загружена новая версия

В popup расширения показывается его версия, например:

```text
v0.1.1
```

Это полезно после `Reload`, чтобы проверить, что Chrome действительно взял новую сборку.

## Как расширение понимает, что вкладка поддерживается

Платформа выставляет bridge-маркер в DOM и отвечает на запросы расширения:

- ping
- export domain

Если активная вкладка поддерживается, на action-кнопке расширения появляется badge `GEN`.

## Как пользоваться

1. Открой вкладку платформы
2. Убедись, что listener уже запущен
3. Нажми на расширение
4. Проверь статус вкладки и listener
5. Укажи абсолютный путь до проекта
6. Нажми `Сгенерировать`

## Что отправляется в listener

Расширение передает bundle с:

- `projectId`
- `environment`
- `sourceUrl`
- `exportedAt`
- `domain`

То есть утилита не ходит в базу напрямую, а получает уже подготовленный dump из открытой админки.
