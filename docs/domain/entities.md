# Сущности Endge

Сущность Endge — сохраняемый документ домена с типом и identity. Ниже собраны основные сущности без разделения на отдельные короткие страницы.

## Структура и контекст

| Сущность | Для чего нужна |
| --- | --- |
| Project | Граница приложения и контейнер его конфигурации |
| Environment | Набор значений для конкретного окружения: переменные, адреса и профили |
| Type | Описание структуры данных и полей |
| Folder | Группировка документов в дереве конфигуратора |
| Version | Снимок или версия конфигурации домена |
| Policy | Правила доступа и другие политики проекта |
| Style | Переиспользуемое описание стилей |

## Данные и преобразования

| Сущность | Для чего нужна |
| --- | --- |
| Query | Получение данных и формирование именованных outputs |
| DataView | Декларативное преобразование входных данных |
| Filter | Условия ограничения выборки и их входные значения |
| Parameter | Типизированное значение конфигурации или пользовательского ввода |
| Converter | Преобразование значения между форматами или типами |
| Computation | Граф вычислений с одним итоговым результатом |
| Store | Материализованное runtime-хранилище, подключаемое через композицию |

## Поведение и интерфейс

| Сущность | Для чего нужна |
| --- | --- |
| ComponentSFC | Исполняемый компонент с портами, шаблоном, логикой и стилями |
| Component | Legacy-документ старого Table/DSL-представления; сам не исполняется |
| Composition | Связь outputs, store и потребителей в единый поток данных |
| Action | Описание последовательности действий и переходов |
| Page и Navigation | Структура экранов и переходов приложения |
| Integration | Настройки взаимодействия с внешней системой или API |
| AuthProfile | Профиль авторизации, используемый запросами и интеграциями |

## Важное различие

Документ — это сохраняемое описание. Исполняемым объектом становится скомпилированная программа и созданный для неё runtime-host. Подробнее: [доменные и runtime-сущности](/architecture/domain-runtime-entities).
