# Сущности Endge

Сущности Endge — это сохраняемые элементы модели приложения. Они задают структуру данных, правила, поведение, представления и связи, из которых затем собирается исполняемая система. Эта страница даёт общую карту домена, а подробные страницы отдельных сущностей расположены ниже в том же разделе.

## Структура и контекст

| Сущность | Для чего нужна |
| --- | --- |
| Project | Граница приложения и контейнер его конфигурации |
| Environment | Набор значений для конкретного окружения: переменные, адреса и профили |
| Type | Описание структуры данных и полей |
| Folder | Группировка документов в дереве конфигуратора |
| Version | Снимок или версия конфигурации домена |
| Policy | Правила доступа и другие политики проекта |
| Style | Переиспользуемое описание стилей |

## Данные и преобразования

| Сущность | Для чего нужна |
| --- | --- |
| [Query](/reference/query) | Получение данных и формирование именованных outputs |
| [Mock data](/reference/mock) | Сохраняемый JSON/text fixture или ссылка на code provider для preview и тестовых данных |
| [DataView](/reference/data-view) | Декларативное преобразование входных данных |
| Filter | Условия ограничения выборки и их входные значения |
| Parameter | Типизированное значение конфигурации или пользовательского ввода |
| [Converter](/reference/converter) | Преобразование значения между форматами или типами |
| [Computation](/reference/computation) | Граф вычислений с одним итоговым результатом |
| Store | Материализованное runtime-хранилище, подключаемое через композицию |
| [I18n Bundle](/reference/i18n-bundle) | Отдельный документ переводов для нескольких локалей |

## Поведение и интерфейс

| Сущность | Для чего нужна |
| --- | --- |
| [ComponentSFC](/reference/component-sfc) | Исполняемый компонент с портами, шаблоном, логикой и стилями |
| Component | Legacy-документ старого Table/DSL-представления; сам не исполняется |
| [Composition](/reference/composition) | Связь outputs, store и потребителей в единый поток данных |
| Action | Описание последовательности действий и переходов |
| Page и Navigation | Структура экранов и переходов приложения |
| Integration | Настройки взаимодействия с внешней системой или API |
| AuthProfile | Профиль авторизации, используемый запросами и интеграциями |

## Важное различие

Документ — это сохраняемое описание. Исполняемым объектом становится скомпилированная программа и созданный для неё runtime-host. Подробнее: [доменные и runtime-сущности](/architecture/domain-runtime-entities).

Общий декларативный API, который используется несколькими сущностями, собран отдельно: [функциональные выражения Endge](/reference/value-expressions). На страницах конкретных сущностей описаны их собственные readers, структурные конструкции и ограничения.

Статическая конфигурация документов и внутренних source-узлов оформляется через
versioned namespaces. Persisted `meta` и публичная compiled metadata имеют
разное назначение: [общий контракт metadata](/reference/metadata).
